Responsive websites are all the rage. The ability to have one codebase that looks good on a computer, tablet, and smartphone is great. In fact, if your app is mainly content it's better to have a responsive website than a native mobile app. Using an existing framework to give you responsive out of the box can be really useful. The popular frameworks at the moment are Twitter Bootstrap and Zurb Foundation. We've used both in various projects recently and the latest Railscast is on Foundation so we watched it this week to see if there were bits we had missed.

One of the key features of Foundation 4 is that it's "mobile first" so it's easier to get it looking good on a smartphone. Because older browsers, including Internet Explorer 8, don't support responsiveness, "mobile first" does mean those browsers have that same mobile look on a bigger screen, but even Microsoft says it's time to upgrade. Other impressions from using both Foundation and Bootstrap is that Foundation is easier to customize, which is important when you make fancy looking websites!

One of the great bits we did pick up from the video is the handy setup command for when you first add Foundation to a project. That'll make things a lot easier next time!
